За основу взята публикация //infostart.ru/public/1276725/. В ней рассмотрен пример обмена через API с Честным знаком с использованием электронной подписи. 

В примере //infostart.ru/public/1276725/ использовались:

- метод запроса информации по коду маркировки (коду идентификации КИ): facade/cis/cis_list
(На данный момент метод является устаревшим и удален из документации "Описание True API" (Версия 331.0). Актуальную документацию можно скачать в личном кабинете Честного знака в разделе "Помощь". )

- устаревшие url адреса.

В данной публикации сделан запрос кодов маркировки группы товаров "Табачная продукция" по периоду выпуска кодов (эмиссии). Информация по дате эмиссии отображена в личном кабинете честного знака в разделе "Коды маркировки".  Код маркировки с последней датой эмиссии всегда отображен первым (верхним). Сортировка идёт по убыванию даты эмиссии. Таким же образом система "отдаёт" и список марок на запрос API.

Наши действия:

1. Создаем POST запрос /api/v4/true-api/cises/search - метод получения информации о списке КИ по заданным фильтрам (см. Описание True API" (Версия 331.0) ).

2. Получаем ответ. Система "отдаст" только одну страницу найденных кодов маркировки (по умолчанию 100 шт., задаётся  параметрами пагинации).

3. Выгружаем ответ в таблицу КМ (Коды Маркировки).

4. Отправляем повторный запрос с изменёнными параметрами пагинации, которые указывают на последний КМ уже находящийся в таблице выгрузки КМ: lastEmissionDate - дата этого КМ и sgtin - сам код маркировки. Система "поймёт", что нужно "отдать" следующую страницу данных. 

5. Получаем ответ.

6. выгружаем в таблицу КМ.

7. Повторяем операцию с п.4

Обработка дополнена запросом списания кодов маркировки. Обновление (18.09.2023 г).

Используем метод POST /lk/documents/create?pg=tobacco - единый метод создания документов (Описание True API Версия 338.0, стр. 62 ). Метод не планируется снимать с поддержки, он используется вместо устаревающего метода.

pg - обязательный параметр, товарная группа (tobacco - табачная продукция, otp - альтернативная табачная продукция, ncp - никотиносодержащая продукция и т.д.)

Пример тела запроса:
{
     "document_format":"string",                                                      - тип документа  - MANUAL – формат * .json,
     "product_document":"<Документ в формате base64>",              
     "type":"string",                                                                         - код типа документа WRITE_OFF - Выбытие, 
      "signature":"<Открепленная УКЭП формата Base64>"              
}

Основной параметр тела запроса  product_document - документ выбытия. 

"Выбытие (для подачи сведений через Единый метод создания документов)" (см. документацию Описание True API Версия 338.0, стр. 292). Подача сведений доступна для всех товарных групп, кроме товарных групп «Морепродукты», «Товары из натурального меха».

 

Что мы делаем:

1. формируем структуру документа "Выбытие" ВыбытиеДокументСтруктура (для наглядности документ можно создать в личном кабинете 

    Честного знака):

    - participantId                   ( ИНН участника оборота товаров),

    - dropoutReason                ( причина списания: Брак "DEFECT", Утрата "DAMAGE_LOSS", Прочее "OTHER" и т.д.),

    - address                          ( адрес списания - строка, адрес организации или магазина, желательно, чтобы адрес был проверен в

                                              личном кабинете Честного знака при ручном создании документа),

    - withChild                        ( признак списания всех вложенных элементов (необязательный параметр, по-умолчанию "true")),

    - sntins                             ( массив кодов маркировки),

    - sourceDocDate                ( дата документа выбытия в формате yyyy-MM-dd ),

    - sourceDocNum                ( номер документа выбытия),

    - sourceDocType                ( тип документа выбытия : "DESTRUCTION_ACT" - "Акт уничтожения" или "OTHER" - "Другой",

                                              необязательный параметр, по-умолчанию "DESTRUCTION_ACT" - акт уничтожения),

    - sourceDocName              ( имя документа - строка, например "Списание товаров", заполняется если тип документа выбытия "OTHER")

2. преобразуем структуру в формат JSON           : ВыбытиеДокумент_JSON,

3. шифруем ВыбытиеДокумент_JSON в base64    : ВыбытиеДокумент_JSON_Base64,

4. формируем откреплённую подпись в формате base64 : ОткрепленнаяПодпись (для получения используется процедура                         

   из публикация //infostart.ru/public/1276725/ ПодписатьТекст(ВыбытиеДокумент_JSON_Base64,Сертификат,Истина) с последним 

   параметром "Истина",

3. формируем структуру тела запроса  ДанныеДляЗапроса:

    - document_format                  ( тип документа "MANUAL" - .json, возможен XML, CSV),

    - product_document                ( тело документа ВыбытиеДокумент_JSON_Base64      ),

    - type                                     ( код типа документа: Выбытие "WRITE_OFF"             ),

    - signature                              ( ОткрепленнаяПодпись - откреплённая подпись (УКЭП) в формате base64   ),

4. преобразуем структуру тела запроса в формат JSON: ДанныеДляЗапроса_JSON,

5. отправляем запрос,
